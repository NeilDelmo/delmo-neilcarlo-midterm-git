## Midterm Examination Reflection

1. **What is Git used for?**
Git is a local version control system used to track changes made to code files over time. It allows developers to revert to previous versions of their project and maintain a detailed history of their updates. This prevents accidental data loss and keeps project files organized.

2. **What is the difference between Git and GitHub?**
Git is the local command-line tool installed in the computer that tracks and manages your file history. GitHub, on the other hand, is a cloud-based hosting service where you can upload and share those Git repositories online. In short, Git tracks your code locally, while GitHub hosts it on the internet for collaboration.

3. **Why is branching useful?**
Branching allows developers to create isolated copies of a project to test new features or fix bugs safely. It ensures that experimental changes do not break or disrupt the stable, working version of the primary code. Once the new feature is fully tested and complete, it can be seamlessly merged back into the main branch.

4. **What command did you use to upload your local commits to GitHub?**
I used the command `git push origin master` to upload my local repository changes to the remote server. This sends all the verified commits from my local computer directly up to my online GitHub repository. It syncs both platforms so that the latest version of my project is safely backed up in the cloud.

5. **What did you learn from this activity?**
Through this activity, I learned how to create and manage separate development branches to work on features without affecting my main project. I also practiced staging, committing, and merging code locally before syncing it with a remote repository. This workflow showed me how professional software developers structure their daily tasks and protect their codebase.
